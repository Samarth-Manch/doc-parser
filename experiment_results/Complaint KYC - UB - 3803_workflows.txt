INITIATOR WORKFLOW (5 steps)
----------------------------------------------------------------------------------------------------

Step 1: Login: The UBL team logs in to the Manch application using the provided credentials.
  Action Type: login

Step 2: File Upload:
  Action Type: upload

Step 3: The UBL team uploads a bulk file containing all necessary details such as outlet codes, contact information, and any other required data.
  Action Type: upload

Step 4: The system validates the uploaded file for completeness.
  Action Type: upload

Step 5: Below are the data point mentioned for the file upload format -
  Action Type: upload

SPOC WORKFLOW (10 steps)
----------------------------------------------------------------------------------------------------

Step 1: SMS Notification: An SMS containing a link is sent to the SPOCâ€™s mobile number.
  Action Type: notify

Step 2: SMS Reminder: Manch will share the SMS containing a link again to SPOC mobile number on twice a Weekly basis till the KYC is submitted by SPOC.
  Action Type: submit

Step 3: OTP Authentication:
  Action Type: verify

Step 4: The SPOC opens the link and receives an OTP (One-Time Password) for additional authentication.
  Action Type: verify

Step 5: The SPOC enters the OTP, and after successful verification, the Manch platform opens the link.
  Action Type: verify

Step 6: Outlet Code Entry:  The SPOC enters the Outlet Code, which is validated by the Manch system against the code associated with the generated link.
  Action Type: validate

Step 7: Agreement to Policies:

Step 8: Upon accessing the link, the SPOC is prompted to agree to the data privacy policy and terms of conditions.

Step 9: Note:  We will use the same TOS used in the Outlet KYC. If there is any changes, please update the same.
  Action Type: update
  Notes: ['Note:  We will use the same TOS used in the Outlet KYC. If there is any changes, please update the same.']

Step 10: Form Opening: Manch will open the form and SPOC needs to enter the required details as mentioned below

APPROVER WORKFLOW (33 steps)
----------------------------------------------------------------------------------------------------

Step 1: Assignment to Approver: Once all the mandatory fields are entered, validated, and the request is submitted, the system automatically assigns the request to the appropriate approver.
  Action Type: validate

Step 2: Individual Approval:
  Action Type: approve

Step 3: The approval flow for individuals will only be routed to the TSE approvers in two specific cases, followed by the UB team as mentioned below.
  Action Type: approve

Step 4: In all other scenarios, it directly goes to the UB team.

Step 5: Establishment Approval : All requests will it directly goes to the UB team.
  Action Type: approve

Step 6: Approver access: Approvers have two methods to access the transaction, review and act on requests as mentioned below -
  Action Type: approve

Step 7: Email Notification:
  Action Type: notify

Step 8: The Manch system sends an email notification to the approver with a link to review the request.
  Action Type: approve

Step 9: The approver clicks the link to access all request details and can either approve or reject the request based on the provided information.
  Action Type: approve

Step 10: Manch System Login:
  Action Type: login

Step 11: The approver can log in to the Manch system, navigate to the "Pending Requests" section, and review the details.
  Action Type: login

Step 12: After reviewing, the approver can either approve or reject the request.
  Action Type: approve

Step 13: TSE View and Action

Step 14: Log into the Manch system: TSE logs into the Manch system and views all pending requests.
  Action Type: login

Step 15: Open the transaction: TSE opens a transaction to see the details uploaded in the bulk file, and details entered by the SPOC, along with the relevant documents.
  Action Type: upload

Step 16: Approve or Reject the request:
  Action Type: approve

Step 17: If TSE clicks "Approve," they enter an approval comment and submit it. The request then moves to the next level of approver.
  Action Type: approve

Step 18: If TSE clicks "Reject," they select a rejection reason from a dropdown (configured based on UB rejection reasons) and submit it. The request returns to the SPOC for necessary modifications.
  Action Type: reject

Step 19: Resubmit by SPOC : Once the SPOC submits the required details, the request assigns back to TSE, who can then repeat the approval or rejection
  Action Type: approve

Step 20: UB approver View and Action
  Action Type: approve

Step 21: Log into the Manch system: UB approver logs into the Manch system and views all pending requests.
  Action Type: login

Step 22: Open the transaction:

Step 23: UB approver opens a transaction to see the details uploaded in the bulk file, and details entered by the SPOC, along with the relevant documents.
  Action Type: upload

Step 24: .

Step 25: Respond to mandatory question:

Step 26: Individual Use case :

Step 27: UB approver answers the mandatory question: "Are you approving this request?" by selecting either "Yes" or "No" from the dropdown menu.
  Action Type: approve

Step 28: Approve or reject the request:
  Action Type: approve

Step 29: If UB approver clicks "Approve," they enter an approval comment and submit it. The request then moves to Accepted status.
  Action Type: approve

Step 30: If UB approver clicks "Reject," they select a rejection reason from a dropdown (configured based on UB rejection reasons) and submit it. The request returns to the SPOC for necessary modifications.
  Action Type: approve

Step 31: Resubmit by SPOC :
  Action Type: submit

Step 32: Once the SPOC submits the required details, the request assigns back to TSE(In case of individual use cases) or ASM (In case of group use cases) who can then repeat the approval or rejection process as outlined above
  Action Type: approve

Step 33: SMS / Email Template if there are any changes to the existing format
  Action Type: notify
